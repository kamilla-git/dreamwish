# Содержание Wiki DreamWish

## 1. Концепция дизайна (Visual Identity)
Мы отошли от стандартных плоских дизайнов. Весь UI построен на принципе "Скевоморфизм 2.0" и магического реализма:
- **Цвета**: `emerald-950` (#022c22) как база, `gold-400` (#fbbf24) для акцентов.
- **Шрифты**: **Alice** для заголовков (сказочный вид) и **Montserrat** для данных.
- **Анимации**: Использование `canvas-confetti` при успехах и `framer-motion` для плавности.

## 2. Работа Скрейпера (Scraper Logic)
Скрейпер — самая сложная часть бэкенда. Он использует:
- `Playwright` в headless режиме.
- `playwright-stealth` для маскировки под реального пользователя.
- Сложную систему селекторов и регулярных выражений для поиска цен на Ozon и WB, даже если страницы динамические.
- Автоматическую замену thumbnails на полноразмерные изображения для Wildberries.

## 3. Реалтайм система (WebSocket)
Реализована через комнаты (Rooms) в Socket.IO:
- При входе на страницу `/w/[slug]`, клиент подписывается на комнату `wishlist_[slug]`.
- При любом действии (резерв, вклад), сервер делает `emit` в эту комнату.
- Все клиенты в комнате мгновенно получают сигнал и обновляют данные.

## 4. Гайд по миграции на PostgreSQL
Для промышленного запуска:
1. Установите `PostgreSQL`.
2. Создайте базу данных.
3. В `backend/.env` пропишите `DATABASE_URL=postgresql+asyncpg://...`.
4. Запустите сервер, SQLAlchemy сама создаст таблицы.

## 5. Обработка Edge-кейсов
- **Битые фото**: Реализован `onError` хендлер, заменяющий битую ссылку на красивое дефолтное фото леса.
- **Превышение суммы**: Бэкенд проверяет `remaining_amount` и не позволяет внести больше 100%.
- **Безопасность**: Все пароли хешируются через `bcrypt`. Токены JWT имеют срок жизни.
